Movie review app

In the previous lab, we have finished creating models and add the required associations.
In this lab we will implement few custom routes which will help the app to evolve.


We will now start creating some views.

Step 1: Generate scaffold for Movie 
rails generate scaffold Movie name:string description:text releasedate:date

Since we created the model already, rails engine will complain that a model already exists. We will tell it to reuse the existing model.
To do that, enter 'n' when prompted like below.

Step 2: Start your rails server


You can access your C9 url once the server is up and running.

Now lets create a route to access the content of categories table.
Step 3: Generate a controller for Categories
rails generate controller categories

This will create categories_controller.rb file in app/controllers directory and also a folder named 'categories' in app/views folder.

Step 4: Add index method in categories controller
Modify the controller file to look like below:
app/controllers/categories_controller.rb 
class CategoriesController < ApplicationController
    def index
       @categories = Category.all
    end
end


Step 5: Add route in routes.rb
Add the following line in the routes file:
app/config/routes.rb 
get 'categories/' => 'categories#index'


Step 6: Create view file for category
Create a file index.html.erb in app/views/categories with the following content:
app/views/categories/index.html.erb 
<h1>Listing Categories</h1>
<table>
  <thead>
    <tr>
      <th>Category</th>
    </tr>
  </thead>
  <tbody>
    <% @categories.each do |category| %>
      <tr>
        <td><%= category.category_name %></td>
      </tr>
    <% end %>
  </tbody>
</table>


Step 7: You can verify that the new route is created with rake routes

rake routes
